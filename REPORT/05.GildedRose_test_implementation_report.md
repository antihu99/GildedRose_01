# Gilded Rose 테스트 구현 결과 보고서

작성 관점: 시니어 Java QA 엔지니어  
대상 프로젝트: Gilded Rose Java 프로젝트  
기술 스택: Java 21, Maven, JUnit 5

## 1. 작업 요약

`docs/requirements_analysis.md`의 아이템별 품질 변경 규칙을 기준으로 `GildedRoseTest.java`를 테스트 설계 중심으로 재작성했다.

주요 반영 사항:

- Normal / Aged Brie / Backstage Pass / Sulfuras / Conjured 5개 아이템 타입별 테스트 작성
- 각 아이템 타입별 최소 5개 이상의 테스트 케이스 구성
- `@DisplayName`으로 테스트 의도 명시
- Given-When-Then 구조 적용
- 가능한 케이스는 `@ParameterizedTest`와 `@CsvSource`로 구성
- `quality` 0, 50 및 `sellIn` 0, -1 경계값 포함
- `sellIn`과 `quality`를 함께 검증

## 2. 수정 파일

### `src/test/java/com/gildedrose/GildedRoseTest.java`

기존 샘플 테스트를 제거하고 아이템 타입별 중첩 테스트 클래스로 재구성했다.

테스트 구성:

- Normal: 5개 케이스
- Aged Brie: 5개 케이스
- Backstage Pass: 7개 케이스
- Sulfuras: 5개 케이스
- Conjured: 6개 케이스

총 테스트 케이스 수: 28개

### `src/main/java/com/gildedrose/GildedRose.java`

테스트 Green 달성을 위해 Conjured 요구사항을 구현하고 기존 조건문을 의미 단위 메서드로 정리했다.

반영 규칙:

- Normal: 판매 기한 전 1 감소, 만료 후 2 감소
- Aged Brie: 판매 기한 전 1 증가, 만료 후 2 증가, 최대 50
- Backstage Pass: 기간 구간별 1/2/3 증가, 공연 이후 0
- Sulfuras: `sellIn`, `quality` 모두 불변
- Conjured: 판매 기한 전 2 감소, 만료 후 4 감소, 최소 0

## 3. 경계값 검증

검증한 필수 경계값:

- `quality = 0`: Normal / Conjured 하한 유지
- `quality = 50`: Aged Brie / Backstage Pass 상한 유지
- `sellIn = 0`: 업데이트 후 `-1`이 되고 만료 규칙 적용
- `sellIn = -1`: 이미 만료된 상태에서 만료 후 규칙 지속 적용
- Sulfuras 예외: `quality = 80`도 변경 없이 유지

## 5. 참고 사항

현재 `pom.xml`의 Maven compiler 설정은 `source/target 1.8`로 되어 있어 Java 21 환경에서 obsolete warning이 출력된다. 테스트 실행에는 영향이 없었으며, Java 21 프로젝트 기준으로 정리하려면 compiler 설정을 별도 작업으로 갱신하는 것이 좋다.
