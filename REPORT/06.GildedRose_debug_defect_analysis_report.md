# Gilded Rose 디버깅 및 결함 분석 보고서

작성 관점: Java QA 엔지니어  
대상 파일: `src/test/java/com/gildedrose/GildedRoseTest.java`, `src/main/java/com/gildedrose/GildedRose.java`  
검증 명령: `mvn test`

## 1. 분석 요약

사용자가 요청한 `mvn test` 실패 로그 기반 결함 분석을 수행했다. 다만 현재 대화에는 실패 로그가 첨부되지 않았고, 로컬에서 `mvn test`를 실행한 결과 전체 테스트가 Green 상태로 확인되었다.

실행 결과:

```text
Tests run: 28, Failures: 0, Errors: 0, Skipped: 0
BUILD SUCCESS
```

따라서 현재 코드 기준으로는 실패한 테스트 케이스의 기대값/실제값 차이를 특정할 수 없으며, 즉시 수정해야 할 재현 가능한 결함은 확인되지 않았다.

## 2. 실패 원인 요약

현재 재현 결과:

- 실패 테스트: 없음
- 기대값/실제값 차이: 없음
- 테스트 상태: Green
- 빌드 상태: Success

`GildedRoseTest.java`는 Normal, Aged Brie, Backstage Pass, Sulfuras, Conjured 아이템에 대해 총 28개 케이스를 검증하고 있으며, 현재 `GildedRose.java` 구현은 해당 테스트 기대값을 모두 만족한다.

## 3. updateQuality() 결함 위치 특정

현재 테스트 실행 결과만 기준으로 하면 결함 위치는 특정되지 않는다.

검토 대상 메서드:

- `GildedRose.java`
- `updateQuality()`
- `updateItem(Item item)`
- `updateBackstagePass(Item item, int startingSellIn)`
- `increaseQuality(Item item, int amount)`
- `decreaseQuality(Item item, int amount)`

현재 구현은 다음 규칙을 만족한다.

- Sulfuras는 `sellIn`, `quality` 모두 변경하지 않는다.
- 일반 아이템은 판매 기한 전 1, 만료 후 2 감소한다.
- Aged Brie는 판매 기한 전 1, 만료 후 2 증가하고 최대 50을 넘지 않는다.
- Backstage Pass는 남은 기간에 따라 1/2/3 증가하고 공연 이후 0이 된다.
- Conjured는 일반 아이템보다 두 배 빠르게 감소한다.
- `quality`는 Sulfuras 예외를 제외하고 0 이상 50 이하로 유지된다.

## 4. 결함 심각도 분류

심각도: Info

근거:

- 현재 `mvn test`가 성공한다.
- 실패 로그가 제공되지 않아 재현 가능한 결함 증거가 없다.
- 제품 동작 위반 또는 테스트 실패가 확인되지 않았다.
- 코드 변경 없이도 현재 테스트 기준 품질 규칙을 만족한다.

실패 로그가 추가로 제공되어 특정 케이스가 재현된다면 심각도는 영향 범위에 따라 재분류해야 한다.

예상 분류 기준:

- Critical: 다수 핵심 아이템 규칙이 깨져 전체 품질 업데이트가 신뢰 불가한 경우
- Major: 특정 아이템 타입의 핵심 비즈니스 규칙이 잘못 적용되는 경우
- Minor: 경계값 일부 또는 메시지/표현 수준의 제한적 문제
- Info: 결함이 재현되지 않거나 코드 변경이 필요 없는 관찰 사항

실패 로그가 있는 경우 다음 정보를 함께 확인해야 한다.

- 실패한 테스트 클래스와 메서드명
- 기대값과 실제값
- 실패한 아이템 이름, `sellIn`, `quality` 초기값
- 업데이트 후 기대 `sellIn`, `quality`
- `GildedRose.java`의 해당 분기 조건
