- SRP/OCP 위반 지점과 근거
- Long Method, Magic Number, Duplicated Code, 복잡도 등 Code Smell
- 리팩토링 우선순위 1~5와 이유
- 요청한 Markdown 표 형식

결과 파일은 `docs/code_quality_report.md`로 생성했다.

### Result

- 생성 파일: `docs/code_quality_report.md`
- 주요 판단: `updateQuality()`가 아이템 순회, 타입 판별, 품질 변경, 판매기한 감소, 만료 후 처리까지 모두 담당하므로 SRP 위반과 Long Method가 핵심 문제로 분석됨
