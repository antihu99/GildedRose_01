구현 방향:
- `TexttestFixture`의 출력 생성을 `generateReport(int days)` / `printReport(PrintStream out, int days)`로 분리
- 기준 출력을 `src/test/resources/golden_master_expected.txt`에 저장
- `GoldenMasterTest`에서 actual 출력과 기준 파일을 문자열 비교
- 줄바꿈 차이 및 UTF-8 BOM 정규화 적용
- `pom.xml`에 Java 21 컴파일 설정과 Surefire JUnit 5 실행 설정 추가

수정·추가 파일:
- `src/test/java/com/gildedrose/TexttestFixture.java`
- `src/test/java/com/gildedrose/GoldenMasterTest.java`
- `src/test/resources/golden_master_expected.txt`
- `pom.xml`

기준 파일 생성:
- `TexttestFixture 30` 실행 결과를 day 0~30(총 31일) 기준으로 고정
- PowerShell에서 UTF-8 BOM 없이 재생성 가능

실행 확인:

```bash
mvn test
```

```text
Tests run: 29, Failures: 0, Errors: 0, Skipped: 0
BUILD SUCCESS
```

기준 파일 재생성 예시:

```powershell
mvn -q -DskipTests test-compile
$text = & java -cp "target/classes;target/test-classes" com.gildedrose.TexttestFixture 30 | Out-String
[System.IO.File]::WriteAllText("src/test/resources/golden_master_expected.txt", $text, [System.Text.UTF8Encoding]::new($false))
```

Approvals 적용 가능성:
- `pom.xml`에 `approvaltests` 의존성이 있으므로 `Approvals.verify(TexttestFixture.generateReport(31));` 형태로 전환 가능
- 이번 구현은 CI 안정성과 기준 파일 위치 명확성을 위해 명시적 파일 비교 방식을 선택
